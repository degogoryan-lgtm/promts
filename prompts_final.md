# Система промптов для генерации артефактов аналитики
## Инвест. платформа (ЛК Брокера)

---

## Описание значимости артефакта

| Раздел | Содержание |
|---|---|
| **Процесс и контекст использования** | Система промптов используется в рамках процесса создания аналитических артефактов для финтех-проекта Инвест. платформа (ЛК Брокера). Каждый промпт создан под конкретную роль — продуктовый аналитик берёт промпт для BRD, системный архитектор для Sequence, DBA для ER, интегратор для OAS. Промпты используются последовательно — результат предыдущего передаётся как контекст в следующий. |
| **Цель создания** | Дать каждой роли точную инструкцию для генерации артефакта. Промпт — это не просто набор слов, а структурированное техническое задание для LLM. От формулировки зависит насколько полезным и релевантным будет результат. Нечёткий запрос приводит к неточным или ошибочным артефактам. Также промпты автоматизируют рутинную часть создания документов. |
| **Что становится определено** | Роль LLM при генерации каждого артефакта. Контекст домена — лимиты, валюты, статусы, типы комиссий. Формат вывода — Markdown, код для sequencediagram.org, XML для draw.io, YAML. Порядок передачи контекста между артефактами. |
| **Пользователи артефакта** | Продуктовый аналитик — использует промпт для генерации BRD. Системный архитектор — для Sequence-диаграмм. DBA — для ER-диаграммы. Интегратор — для OAS. Каждый берёт свой промпт и вставляет в DeepSeek. |
| **Использование в дальнейшем** | После прогона всех промптов через DeepSeek получается комплект согласованных артефактов — BRD, Sequence, ER, OAS. Сущности из ER попадают в OAS, шаги из Sequence отражены в BRD. Артефакты размещаются в GitHub и подключаются к Confluence через MacroPack. |
| **Последствия отсутствия** | Без системы промптов каждая роль делает артефакт самостоятельно без единой инструкции. Продуктовый аналитик, архитектор, DBA и интегратор принимают решения независимо — возникает разнобой между документами. Сущности в ER не совпадают с OAS, шаги в Sequence не отражены в BRD. На приёмке заказчик говорит «я имел в виду другое» — и переделывать придётся всё. |

## Контекст проекта

Все артефакты создаются для функциональности
финтех-проекта Инвест. платформа (ЛК Брокера).

---

## Как пользоваться документом

Промпты используются последовательно —
каждый следующий опирается на результат предыдущего.

Порядок работы:
1. Отправь промпт в DeepSeek
2. Дождись Шага 1 — проверь результат
3. Напиши "Подтверждаю" только если всё верно
4. Получи Шаг 2 — сохрани результат
5. Вставь сохранённый результат в следующий промпт

### 1.1 Цель документа
Описать бизнес-требования к созданию Личного кабинета брокера

|---|---|---|
| 1 | Инвестор не знает реальную сумму комиссии до сделки | Платформа рассчитывает комиссию до подтверждения заявки |
| 2 | Вывод средств занимает 2–3 дня без отслеживания статуса | Заявка на вывод в реальном времени со статусами, срок до 1 дня |

| 5 | Нельзя отменить ошибочную заявку | Отмена заявки в статусе PENDING или PROCESSING |

| № | Название | Описание | Приоритет |
|---|---|---|---|
| BR-1 | Создание рыночной заявки | Инвестор создаёт заявку по текущей рыночной цене | Высокий |
| BR-2 | Создание лимитной заявки | Инвестор выставляет лимитную заявку по указанной цене | Высокий |
| BR-3 | Ограничение суммы сделки | Сумма заявки не превышает 10 000 000 рублей | Высокий |
| BR-4 | Предварительный расчёт комиссии | До подтверждения система показывает сумму комиссии | Высокий |
| BR-5 | Отмена заявки | Инвестор отменяет заявку в статусе PENDING или PROCESSING | Высокий |
